# Red Flags — Agent-Framing

Watch for these during sessions. Any one of them signals the framing is broken.

- A scope that lists fewer than 6-8 dimensions covered. Real competitive scopes have 30+ dimensions.
- A "phase 2" or "future ambitions" parking lot containing capabilities competitors have today.
- An eval design that selects questions favoring our system's strengths rather than testing the full competitive surface.
- An audit's findings being treated as the complete defect list.
- "We can defer X" without an explicit competitive comparison showing X is a future ambition (not a deficit).
- A planning session that doesn't reference a single SOTA framework or competitive system by name.
- Working from inside one system's mental model without questioning whether the model is right.
- Session-start work without a `git fetch origin && git log` competitive sweep.

When a red flag fires: STOP the session direction and re-scope from [README.md](README.md) + [12-axes.md](12-axes.md).
